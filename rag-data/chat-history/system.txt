🧱 CORE PRODUCT CONCEPT

VestingApp is a SaaS platform where project owners pay $99/month in USDC (on Solana) to manage token vesting schedules for their users.

The system is:
- Non-custodial (tokens are not held centrally)
- Subscription-based (monthly recurring)
- Webhook-driven (Helius detects payments)
- Fully automated (retry system + cron jobs)
- Real-time monitored (analytics dashboard)

--------------------------------------------------

🏗️ SYSTEM ARCHITECTURE

1. USER FLOW

- User connects wallet (Phantom)
- User pays 99 USDC to BUSINESS_WALLET
- User includes memo:
  vestingapp-starter-<userIdPrefix>

- Helius detects transaction
- Webhook fires → supabase/functions/helius-webhook
- Backend validates:
  - correct token (USDC mint)
  - correct receiver (business wallet)
  - correct amount (>= 99 USDC)
  - valid memo format

- Backend finds user via prefix
- Subscription is inserted into database
- processed_transactions prevents duplicates

--------------------------------------------------

3. WEBHOOK SYSTEM

File:
supabase/functions/helius-webhook/index.ts

Responsibilities:
- Validate Helius webhook request (Authorization header)
- Parse transaction array
- Loop through tokenTransfers
- Validate:
  - mint === USDC_MINT
  - toUserAccount === BUSINESS_WALLET
  - tokenAmount >= REQUIRED_AMOUNT

- Extract memo:
  vestingapp-starter-<prefix>

- Find owner via:
  project_owners.ilike(prefix%)

- Atomic insert into processed_transactions (lock)
- Prevent duplicate processing

- Insert subscription if valid
- Update processed_transactions:
  status = completed

--------------------------------------------------

5. RETRY SYSTEM

File:
supabase/functions/retry-webhooks/index.ts

Flow:
- Cron triggers function
- Fetch failed transactions from processed_transactions
- Replay original webhook payload
- Increment retry_count
- Stop after max retries
- Update status:
  failed → retrying → completed

Important:
- Uses HELIUS_SECRET internally
- Calls helius-webhook again

--------------------------------------------------

6. CRON JOBS

retry-webhooks:
- runs periodically
